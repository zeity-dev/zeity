## Overview

Zeity can be self-hosted on your own infrastructure, giving you full control over your data and customization options. This guide covers different deployment methods using Docker.

## Prerequisites

Before you begin, ensure you have:

- **Docker** 24+ installed ([Install Docker](https://docs.docker.com/get-docker/))
- **Docker Compose** 2.0+ (included with Docker Desktop)
- **PostgreSQL** 14+ database (or use Docker)
- At least **1GB RAM** and **2GB disk space**

## Quick Start with Docker

The fastest way to get Zeity running is with Docker.

### 1. Pull the Image

```bash
docker pull ghcr.io/zeity-dev/zeity:latest
```

### 2. Run with Docker

```bash
docker run -d \
  --name zeity \
  -p 3000:3000 \
  -e ZEITY_DATABASE_URL="postgresql://user:pass@host:5432/zeity" \
  -e ZEITY_SESSION_PASSWORD="your-secret-key-min-32-chars" \
  ghcr.io/zeity-dev/zeity:latest
```

Visit [http://localhost:3000](http://localhost:3000) ðŸŽ‰

::warning
This setup uses an external database. Make sure PostgreSQL is accessible.
::

## Docker Compose (Recommended)

For a complete setup with database and optional services, use Docker Compose.

### 1. Create docker-compose.yml

```yaml [docker-compose.yml]
services:
  postgres:
    image: postgres:18
    restart: unless-stopped
    environment:
      POSTGRES_USER: zeity
      POSTGRES_PASSWORD: zeity_password
      POSTGRES_DB: zeity
    volumes:
      - postgres-data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U zeity"]
      interval: 10s
      timeout: 5s
      retries: 5

  zeity:
    image: ghcr.io/zeity-dev/zeity:latest
    restart: unless-stopped
    depends_on:
      postgres:
        condition: service_healthy
    ports:
      - "3000:3000"
    environment:
      ZEITY_DATABASE_URL: postgresql://zeity:zeity_password@postgres:5432/zeity
      ZEITY_SESSION_PASSWORD: change-this-to-random-string-min-32-chars
      ZEITY_JWT_SECRET: change-this-to-random-string-min-32-chars
      NODE_ENV: production
      # Optional: SMTP for emails
      # ZEITY_MAILER_SMTP_HOST: smtp.example.com
      # ZEITY_MAILER_SMTP_PORT: 587
      # ZEITY_MAILER_SMTP_USER: your-email@gmail.com
      # ZEITY_MAILER_SMTP_PASS: your-app-password
      # Optional: S3-compatible storage
      # ZEITY_S3_END_POINT: https://s3.example.com
      # ZEITY_S3_ACCESSKEY: your-access-key
      # ZEITY_S3_SECRETKEY: your-secret-key
      # ZEITY_S3_BUCKET: zeity-uploads

volumes:
  postgres-data:
```

### 2. Start Services

```bash
docker-compose up -d
```

### 3. Check Status

```bash
docker-compose ps
docker-compose logs -f zeity
```

## Environment Variables

### Required

| Variable | Description | Example |
|----------|-------------|---------|
| `ZEITY_DATABASE_URL` | PostgreSQL connection string | `postgresql://user:pass@host:5432/zeity` |

### Recommended (Production)

| Variable | Description | Example |
|----------|-------------|---------|
| `ZEITY_SESSION_PASSWORD` | Session encryption key (32+ chars) | Generate with `openssl rand -base64 32` |
| `ZEITY_JWT_SECRET` | JWT token signing secret | Generate with `openssl rand -base64 32` |

::warning
While not mandatory, `ZEITY_SESSION_PASSWORD` and `ZEITY_JWT_SECRET` are **highly recommended for production** to ensure secure authentication and session handling.
::

### Optional - Email (SMTP)

| Variable | Description | Default |
|----------|-------------|---------|
| `ZEITY_MAILER_SMTP_HOST` | SMTP server hostname | - |
| `ZEITY_MAILER_SMTP_PORT` | SMTP server port | `587` |
| `ZEITY_MAILER_SMTP_USER` | SMTP username | - |
| `ZEITY_MAILER_SMTP_PASS` | SMTP password | - |
| `ZEITY_MAILER_FROM` | Email sender address | `noreply@example.com` |

### Optional - File Storage (S3)

| Variable | Description | Default |
|----------|-------------|---------|
| `ZEITY_S3_END_POINT` | S3-compatible endpoint | - |
| `ZEITY_S3_ACCESSKEY` | S3 access key | - |
| `ZEITY_S3_SECRETKEY` | S3 secret key | - |
| `ZEITY_S3_BUCKET` | S3 bucket name | - |

::tip
Zeity supports any S3-compatible storage solution, including:
- **RustFS** - Lightweight S3-compatible object storage
- **MinIO** - Self-hosted S3-compatible object storage
- **Hetzner Object Storage** - Hetzner's S3-compatible storage solution
- **OVH Object Storage** - OVH's S3-compatible storage solution
- **AWS S3** - Amazon's object storage service
Without S3 configuration, files will be stored locally in the container.
::

### Optional - OAuth Providers

| Variable | Description |
|----------|-------------|
| `NUXT_OAUTH_GOOGLE_CLIENT_ID` | Google OAuth Client ID |
| `NUXT_OAUTH_GOOGLE_CLIENT_SECRET` | Google OAuth Client Secret |

| `NUXT_OAUTH_MICROSOFT_CLIENT_SECRET` | Microsoft OAuth Client Secret |

## Updates

### Update to Latest Version

```bash
# Pull latest image
docker-compose pull zeity

# Restart with new image
docker-compose up -d zeity
```

### Check Current Version

```bash
docker exec zeity cat /home/node/package.json | grep version
```

## Troubleshooting

### Container Won't Start

**Check logs:**
```bash
docker-compose logs zeity
```

**Common issues:**
- Database not ready: Wait for PostgreSQL health check
- Invalid environment variables: Check `ZEITY_SESSION_PASSWORD` length (min 32 chars)
- Port already in use: Change port mapping `3001:3000`

### Database Connection Failed

**Verify database is running:**
```bash
docker-compose ps postgres
```

**Test connection:**
```bash
docker exec postgres pg_isready -U zeity
```

**Check connection string:**
- Format: `postgresql://user:password@host:port/database`
- Ensure credentials match PostgreSQL environment variables

### Out of Memory

**Increase container memory:**
```yaml
services:
  zeity:
    deploy:
      resources:
        limits:
          memory: 2G
```

### Performance Issues

**Database optimization:**
```bash
docker exec postgres psql -U zeity -c "VACUUM ANALYZE;"
```

**Monitor resource usage:**
```bash
docker stats zeity postgres
```

## Security Best Practices

### 1. Use Strong Secrets

```bash
# Generate secure session password
openssl rand -base64 32

# Generate for environment file
echo "ZEITY_SESSION_PASSWORD=$(openssl rand -base64 32)" >> .env
```

### 2. Regular Updates

```bash
# Update Docker images weekly
docker-compose pull
docker-compose up -d
```

### 3. Use .env File

Create `.env` file for sensitive data:

```bash [.env]
ZEITY_DATABASE_URL=postgresql://zeity:secret@postgres:5432/zeity
ZEITY_SESSION_PASSWORD=your-very-long-random-secret-key-here
ZEITY_MAILER_SMTP_PASS=smtp-password
ZEITY_S3_SECRETKEY=s3-secret-key
```

Reference in docker-compose.yml:

```yaml
services:
  zeity:
    env_file: .env
```

### 5. Restrict Database Access

```yaml
services:
  postgres:
    # Don't expose port externally in production
    # ports:
    #   - "5432:5432"
```
